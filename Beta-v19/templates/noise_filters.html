SOC Recommendations — Log Optimisation
Wazuh Server-Side (Highest Impact)
  • Add rule_id 4804 to /var/ossec/etc/rules/local_rules.xml with <overwrite> to set no_alert: yes — eliminates 295k/day at source
  • Set min_severity in your Wazuh filebeat/logstash pipeline to only forward events with rule.level ≥ 7
  • For SonicWall: use Wazuh localfile decoder overrides to raise the minimum level for IKEv2 events
  • Configure FIM exclusions in ossec.conf for high-churn paths (temp dirs, Windows update staging)
  • Use Wazuh active-response <group>sonicwall</group> level override to suppress rules 4803/4804
SentinelX Application Layer (Active Now)
  • All 12 default filters above are active — suppressed events will not be stored or shown in dashboards
  • Minimum level filter (level ≥ 7) prevents low-value events from being stored in the database
  • ITDR/XDR engine uses targeted queries and is not affected by FIM noise
  • Add custom filters below to tune further (agent-specific, description-based, or additional rule IDs)
  • Review the Top Alert Sources panel weekly and suppress any new recurring noise rules
Do NOT Suppress
  • 62123 — Windows Defender malware detections (KMSpico/HackTool)
  • 60104 / 60106+ — Windows logon failures & privilege escalation
  • 5501+ / 5710+ — Wazuh agent disconnect events
  • Any rule with level ≥ 12
  • SonicWall rules related to blocked inbound connections (rule 4801)
Expected Outcome
  • Estimated noise reduction: ~92–95% of daily alert volume suppressed
  • Dashboard alert feed will show predominantly level ≥ 7 events
  • MTTR (Mean Time to Respond) improvement: analysts see threats without scrolling through thousands of IKEv2 entries
  • ITDR scans will correlate real behaviour patterns more accurately
  • False negative risk: Low — all suppressed rules are confirmed infrastructure noise
